Challenge 1 ⭐ Beginner Ollama — local, no AWS

Your First Agent

Build a simple AI agent that runs 100% on your machine.

🎯 What is this challenge?

You write the minimum amount of code needed to create an AI agent. No cloud, no API keys, no cost — just Python talking to a local AI model via Ollama. Think of it as "Hello World" for AI agents.

💡 What you learn
  • ✓
    How Ollama runs an AI model locally on your laptop
  • ✓
    What the Strands SDK Agent class is and how to use it
  • ✓
    What a system prompt is — instructions that shape the agent's personality and behaviour
  • ✓
    How to send a question to the agent and read the response

Challenge 2 ⭐⭐ Intermediate Amazon Nova Pro — Bedrock

Adding Tools

Give your agent superpowers — a calculator, weather lookup, and age calculator.

🎯 What is this challenge?

An AI agent by itself can only generate text from its training data. Tools let it actually do things — call APIs, run code, fetch live data. This challenge teaches you to give your agent real capabilities. We also switch from Ollama to Amazon Nova Pro running on AWS Bedrock.

💡 What you learn
  • ✓
    How to use the built-in calculator tool from strands_tools
  • ✓
    How the @tool decorator turns any Python function into an agent tool
  • ✓
    Why the docstring matters — the agent reads it to decide when to use the tool
  • ✓
    How to call a live external API (wttr.in for weather) from inside a tool
  • ✓
    How the agent automatically chooses which tool to use based on your question

Challenge 3 ⭐⭐ Intermediate Amazon Nova Pro — Bedrock

Persistent Memory

Give your agent a brain — it remembers facts even after you restart the program.

🎯 What is this challenge?

By default, every conversation with an AI starts fresh — it has no idea who you are. This challenge adds persistent memory using FAISS (a local vector database). You tell the agent your name once, and it remembers it — even if you close and reopen the program.

🧠 Two types of memory — what's the difference?
Type How it works After restart?
Conversation history Strands sends all previous messages to the model each turn ❌ Lost
mem0_memory (FAISS) Stores facts as vectors on disk, retrieved when relevant ✅ Persists
💡 What you learn
  • ✓
    What a vector store is and why it's used for memory (FAISS stores text as numbers that capture meaning)
  • ✓
    How mem0_memory from strands_tools works as a memory tool
  • ✓
    How to build a multi-turn conversation loop (the agent keeps chatting until you type "quit")
  • ✓
    How the agent automatically decides when to store a memory vs just answer
🔧 How memory storage works
🗣️
"Remember my name is Ravi and I love biryani"
You tell the agent something about yourself
↓
🧠
Agent uses mem0_memory tool
Converts "Ravi loves biryani" to a vector and saves it to disk
↓
❓
"What food do I like?" (new session)
Agent searches memory, finds the biryani fact, answers correctly
📺 Expected output
Terminal

Agent: ✅ I'll remember that! Your name is Ravi

✅ Challenge 3 complete!
💡 Key hints Import mem0_memory from strands_tools and add it to the tools list. The system prompt should explicitly tell the agent to use memory — without that instruction, it may not use the tool consistently.
⭐ Bonus Store 3 different preferences (name, city, favourite food). Then quit the program, restart it, and ask "What do you know about me?" — the agent should recall all three from the FAISS store on disk!
Challenge 4 ⭐⭐⭐ Advanced Amazon Nova Pro — Bedrock

The Full Agent

Combine everything: tools + memory + streaming. The complete picture.

🎯 What is this challenge?

This is the capstone of the fundamentals. You bring together everything from challenges 1–3 and add one more concept: streaming. Instead of waiting for the full response, the agent prints each word as it generates — just like ChatGPT's typing animation. The result is a fully capable, production-quality interactive agent.

💡 What you learn
  • ✓
    How streaming works — a callback function that fires for each token as it arrives
  • ✓
    How to combine multiple tools in one agent (calculator + weather + age + memory)
  • ✓
    How to build a callback handler that also shows which tool is being used in real time
  • ✓
    How all the Strands concepts fit together into a real interactive product
⚡ What streaming looks like

Without streaming, you stare at a blank terminal for 3–5 seconds. With streaming, you see the response build word by word instantly.

Without streaming (old way) → Wait 5 seconds → entire response appears at once
With streaming (this challenge) → Words appear immediately, one by one as the model generates them

The callback_handler parameter on the Agent is what enables this. It's just a function that gets called every time a new token (word piece) arrives.

💡 Key hints The streaming callback function receives an event dict. Check for "data" in the event to print tokens, and "current_tool_use" to show which tool is running. Pass it as callback_handler=your_function to the Agent.
⭐ Bonus Invent a new custom tool — something fun like a random quote generator, a currency converter, or a countdown calculator (how many days until a date). Add it to the agent and test it in conversation!
Challenge 5 🚀 Innovate — Build from scratch Bedrock + MCP Server

MCP-Powered Agent

Connect your agent to the outside world via MCP servers. No starter code — your idea, your build.

🎯 What is this challenge?

MCP (Model Context Protocol) is a standard that lets AI agents connect to external services — like plugging in a USB device. Instead of writing your own tools from scratch, you connect to a pre-built MCP server that already has dozens of capabilities. This challenge is open-ended: you decide what to build. The most creative and useful agent wins a shoutout! 🏆

🔌 What is MCP?
🤖
Your Strands Agent
Asks questions, wants capabilities
↓ MCP Protocol (standard interface)
🔌
MCP Server
A pre-built package with many tools (AWS docs, CDK, GitHub...)
↓
🌐
External data / APIs
Real AWS docs, code repos, pricing data, etc.

MCP is the same standard used by Claude, Cursor, and other major AI tools — learning it now puts you ahead.

💡 What you learn
  • ✓
    What the Model Context Protocol (MCP) is and why the industry is adopting it
  • ✓
    How to connect to an MCP server using MCPClient from Strands
  • ✓
    How to list and use tools exposed by an MCP server automatically
  • ✓
    How to design and build a complete agent from scratch — no starter code
  • ✓
    How to combine MCP tools with memory, streaming, and custom tools
